Appearance

On this page

VitePress带有多种内置Markdown扩展，标题会自动生成锚点，也支持自定义锚点；内部链接会转换为路由链接，外部链接自带特定属性；支持YAML格式的frontmatter、GitHub风格表格、Emoji、目录表、自定义容器等。同时，代码块具备语法高亮、行高亮、聚焦、颜色差异等功能，还能导入代码片段、分组代码块、包含其他Markdown文件，也支持数学方程和图片懒加载，可通过配置文件进行高级配置。

详细总结 ​

标题锚点 ​

VitePress中标题会自动应用锚点，可使用markdown.anchor选项配置锚点渲染。

1.1 自定义锚点：格式为`# 标题内容 {#自定义锚点}`，方便链接引用。

```bash
# 使用自定义锚点 {#my-anchor}
```

链接 ​

  1. 内部链接：会转换为单页导航的路由链接。子目录中的index.md自动转换为index.html，对应/路径。例如在foo/one.md文件中，[Home](/)可导航至根目录index.html 。
  2. 外部链接：自带target="_blank" rel="noreferrer"属性，防止链接被恶意利用。

frontmatter ​

支持YAML格式的frontmatter，如：

yaml
---
title: Blogging Like a Hacker 
lang: en-US
---

表格与Emoji ​

  1. GitHub风格的表格：使用|表头1|表头2|表头3|等语法创建，可设置对齐方式，如| col 3 is | right-aligned | $1600 |。
  2. Emoji：直接输入Emoji代码即可显示，可在特定列表查找支持的Emoji。

目录表与自定义容器 ​

  1. 目录表 (TOC)：输入[[toc]]生成，可通过markdown.toc选项配置呈现效果。

  2. 自定义容器

可定义默认标题和自定义标题，还能在站点配置中全局设置自定义标题，如通过修改config.ts中的markdown.container配置项实现，示例：

md
::: danger
危险区域，请勿继续
:::

::: details 点我查看代码

\`\`\`js
console.log('Hello, VitePress!')
\`\`\`
:::

危险

危险区域，请勿继续

点我查看代码
js
console.log('Hello, VitePress!')

代码块功能 ​

  1. 语法高亮：VitePress使用Shiki实现代码块语法高亮，指定语言别名（如js）即可。

  2. 行高亮：通过在代码块后用{行号}或在代码行添加// [!code highlight]注释实现，支持指定多行、多个单行或混合指定。

  3. 聚焦：使用// [!code focus]注释聚焦某一行，还可通过// [!code focus:<lines>]定义聚焦行数。

  4. 颜色差异：// [!code --]或// [!code --] // [!code ++]注释可创建代码diff效果。

  5. 高亮“错误”和“警告”：// [!code warning]或// [!code error]注释可对代码行相应着色。

  6. 行号：在配置文件中设置markdown.lineNumbers: true启用，也可在代码块用:line-numbers等标记覆盖配置。

  7. 代码导入与包含

    • 导入代码片段：使用<<< @/filepath语法，支持行高亮和从相对路径导入，还可结合VS Code region使用。
    • 代码组：用::: code-group对多个代码块分组，可在组内导入代码片段。
    • 包含markdown文件：使用<!--@include: ./filepath.md-->语法，支持选择行范围，如<!--@include: ./filepath.md{3,}-->。

其他功能 ​

  1. 数学方程：安装markdown-it-mathjax3并在配置文件设置markdown.math: true后可使用，如$a \ne 0$等语法。

  2. 图片懒加载：在配置文件设置markdown.image.lazyLoading: true可启用。

  3. 高级配置：VitePress使用markdown-it作为渲染器，可在.vitepress/config.js中通过markdown选项自定义markdown-it实例，如使用markdownItAnchor等插件。
