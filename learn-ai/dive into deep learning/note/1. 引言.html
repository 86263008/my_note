    4. 疑难问题

    1. 传统编程模式为何难以处理天气预测、图像识别类任务？ 答：这类任务内在规律复杂多变，输入与结果的关联无法被人类完整梳理并编写固定规则，海量计算关系也超出人工编码的能力范畴。
    2. 参数、模型、学习算法三者存在怎样的关联？ 答：参数是可调整的调控变量，调整参数得到不同模型；学习算法依托数据集筛选最优参数，以此确定最终可用模型。
    3. 机器学习与传统程序的核心运行差别是什么？ 答：传统程序依靠预设逻辑执行任务，性能不会自主提升；机器学习依托数据训练迭代优化，任务表现可持续改善。

    6. 核心内容解析

    1. 知识点梳理

    1. 传统编程依靠人工梳理全部业务规则编写代码，仅能执行固定逻辑，无法自主优化迭代，适配场景有限。

    2. 机器学习依托观测数据与环境交互积累经验，自动优化自身性能，可解决人工难以设定规则的复杂任务。

    3. 深度学习隶属于机器学习分支，可赋能计算机视觉、自然语言处理等多个前沿领域。

    4. 以语音唤醒识别为例，人工无法直接编写音频到判定结果的映射代码，机器学习可借助标注数据完成任务建模。

    5. 机器学习依靠参数调整生成模型，通过专属学习算法，基于数据集筛选最优参数组合。

    6. 完整训练流程包含参数初始化、获取标注数据、调整参数优化效果、循环迭代至性能达标四个环节。

    7. 数据编程区别于传统代码编程，以数据驱动确定程序行为，是机器学习的核心实现思路。

    3. 案例逻辑拆解

    案例：语音唤醒词识别

    1. 任务难点：音频每秒海量采样数据，声波信号与唤醒词无直观对应关系，人工无法编写判定规则。

    2. 解决思路：放弃直接编写判定代码，采用数据驱动模式，收集带正负标签的音频数据集。

    3. 建模方式：设定灵活模型框架，依靠参数调控模型行为，划定可适配任务的模型族范围。

    4. 训练迭代：随机初始化参数，不断利用标注样本调整参数，降低预测误差。

    5. 最终效果：训练完成的模型可自主识别指定唤醒词，完成音频输入到判定结果的输出映射。

    7. 归纳总结

    1. 章节主旨

    本部分内容对比传统人工编程的固有缺陷，引出机器学习与深度学习的基础概念，结合日常生活语音唤醒实例，讲解机器学习解决复杂任务的核心思路，阐释参数、模型、算法等基础概念，梳理标准化训练流程，点明数据编程的核心思想，区分传统编码与数据驱动建模的差异，为后续深度学习知识学习搭建基础认知框架。

    2. 各类学习方式异同要点

    1. 相同点：最终目的均为实现指定自动化任务，服务实际生产与生活场景。

    2. 不同点

      • 规则来源：传统编程规则由人工预设编写；机器学习规则由数据自主学习生成。
      • 性能变化：传统程序运行状态固定，无自我提升能力；机器学习可随数据量增加持续优化性能。
      • 适配场景：传统编程适合逻辑清晰、边界明确的事务处理；机器学习擅长规律模糊、计算复杂的感知类任务。
      • 优化方式：人工修改代码完成功能调整；算法迭代参数实现模型优化。

    3. 记忆重点

    1. 核心差异：传统编程人工定规则，机器学习数据学规律。

    2. 基础三要素：参数、模型、学习算法，三者协同完成建模。

    3. 训练四步骤：随机初始化参数→获取标注样本→参数调优→循环迭代达标。

    4. 核心思想：摒弃固定代码逻辑，以数据驱动程序行为。

    4. 学习感悟

    传统编程依靠人类思维梳理逻辑，能力存在明显边界，而机器学习打破了人工编码的局限，借助数据挖掘隐藏规律，能够解决诸多人类难以把控的复杂问题。深度学习作为进阶方向，依托多层表征进一步挖掘数据价值。学习该部分内容后，认识到数据是机器学习的核心根基，后续学习不能局限于代码编写，更要理解数据驱动建模的底层逻 辑，把握模型训练与优化的本质思路。

    1.3.1 监督学习

    总述

    监督学习是工业界应用最广泛的机器学习类型，核心任务是利用带标签的训练数据，学习从输入特征到对应标签的映射关系，实现对未知样本的标签预测。

    1. 数据特点：训练集由大量特征-标签成对的样本构成，标签为任务的预测目标。
    2. 学习流程：① 收集并标注带真实标签的训练数据；② 使用监督学习算法训练得到模型；③ 用训练好的模型对新样本做预测。
    3. 本质：学习估计给定特征下对应标签的条件概率。

    各细分任务总结

    1. 回归

      • 任务定义：标签为连续数值的监督任务，多用于回答“有多少”类问题。
      • 典型场景：房屋估价、电影评分预测、病人住院时长、降雨量预测等。
      • 原理与示例：通过学习特征与数值标签的关联规律做预测，生活中估算维修服务费就是简单的线性回归应用；该类任务常使用平方误差作为损失函数。

    2. 分类

      • 任务定义：标签为离散类别，用于判断样本“属于哪一类”。
      • 主要分类
        1. 二分类：仅两个类别，如区分猫和狗、判断蘑菇是否有毒；
        2. 多分类：类别数量大于两个，如手写数字识别（0~9）。
      • 模型输出：一般输出样本属于每个类别的概率，辅助判断预测结果的置信度；主流损失函数为交叉熵。
      • 拓展类型：层次分类：类别存在层级关系，不同分类错误的代价不同（如将宠物狗误判为同类犬种影响小，误判为动物大类外物种则错误严重）。
      • 实际考量：高风险场景下，不能仅依靠概率最高的类别决策，需结合风险损失综合判断。

    3. 标记问题（多标签分类）

      • 任务特点：区别于传统互斥分类，一个样本可同时对应多个类别标签，标签之间互不排斥。
      • 典型场景：为文章、图片打多个主题标签；对生物医学文献标注专业术语等。

    4. 搜索

      • 核心目标：并非简单分类，而是对海量内容依据相关性排序，顺序直接影响使用效果。
      • 实现思路：先为每条内容计算相关性分数，再按分数高低排序。
      • 发展：早期依靠规则算法（如PageRank），现代搜索引擎结合机器学习与用户行为计算相关性。

    5. 推荐系统

      • 任务目标：根据用户偏好实现个性化内容/商品推送，广泛用于影视、电商、音乐等领域。
      • 反馈类型
        1. 显性反馈：用户主动评分、文字评价；
        2. 隐性反馈：用户点击、跳过、浏览时长等行为数据。
      • 现存问题：评分数据易两极分化；容易形成“越推荐越热门”的反馈循环；数据偏见等仍是待解决的研究问题。

    6. 序列学习

      • 任务特点：处理可变长度的输入/输出序列，要求模型具备“记忆”能力，捕捉序列前后的关联关系，视频、文本、语音、医疗时序数据均属于此类。
      • 常见细分场景
        1. 文本标记与解析：输入、输出序列长度基本一致，为文本标注实体、词性等属性；
        2. 自动语音识别：长音频序列转换为短文本序列；
        3. 文本转语音：短文本序列转换为长音频序列；
        4. 机器翻译：两种不同语言的序列互相转换，语序、长度通常存在差异；
      • 其他应用：用户网页浏览轨迹分析、人机对话系统等。

    1.3.2 无监督学习

    1. 基本定义 与监督学习不同，无监督学习使用的数据集不包含标签，也不会给模型设定明确的预测目标。类比来说，监督学习如同有明确指令指导工作，而无监督学习仅提供原始数据，需要模型自主挖掘数据内部的规律、结构与关联。

    2. 典型任务与应用

      • 聚类：在无标签数据中自动对样本分组。例如将海量照片划分为风景、宠物等类别，或是根据网页浏览行为对用户进行群体划分。
      • 主成分分析：用少量核心参数/维度概括数据的主要特征，实现数据降维。比如提炼物体运动的关键属性、总结人体体型特征，也可用于挖掘词汇间的语义关联。
      • 因果关系与概率图模型：从观测数据中挖掘不同变量之间的内在联系与因果逻辑，例如分析房价、污染、收入、犯罪率等多项数据间的相互影响。
      • 生成对抗网络：无监督学习的重要方向，可学习真实数据的分布规律，进而合成图像、音频等复杂的全新数据。

    1.3.3 与环境互动

    1. 传统学习模式的局限 此前介绍的监督学习、无监督学习都属于离线学习：先从环境中一次性收集完整数据集，再独立完成模型训练，训练过程中算法不再和环境产生交互。这种模式的优势是可以专注于模式识别，但能解决的任务范围十分有限。

    2. 与环境互动的核心特点 真正的智能体不只是做单纯预测，而是会主动和真实环境产生交互；智能体执行的动作会改变环境，环境变化又会进一步影响智能体后续获取的观测结果，形成动态关联。

    3. 该场景下的核心思考问题 围绕智能体与环境的动态关系，衍生出一系列关键问题：

      • 环境是否能记住智能体过往的行为；
      • 环境能否为模型建模提供辅助；
      • 环境是否存在对抗属性（如垃圾邮件拦截、竞技游戏场景）；
      • 环境本身的状态与价值；
      • 环境是否会发生变化（自然演变或受智能体行为影响），这一情况会引发分布偏移，即后续数据与训练数据分布不一致。

    4. 内容衔接 对“智能体与环境交互”的探讨，是引出下一节强化学习的铺垫，强化学习正是专门针对这类动态交互场景设计的机器学习方向。

    1.3.4 强化学习

    强化学习是一类聚焦智能体与环境持续交互的机器学习方向，深度强化学习则是将深度学习融入其中的热门研究领域，深度Q网络、AlphaGo是该领域极具代表性的成果。

    1. 核心交互流程 整个过程形成循环：智能体从环境中获取观察，依据观察做出动作并作用于环境，随后从环境得到奖励，再接收新一轮观察，不断重复该过程。

    2. 核心目标 强化学习的核心并非单次预测，而是学习一套策略（从环境观察到执行动作的映射规则），最终让智能体在长期交互中收获最大化的累积奖励。

    3. 与监督学习的核心区别及特有难题

      • 监督学习依靠明确标签指导学习，而强化学习仅返回奖励，不会直接告知每个观测对应的最优动作；
      • 存在学分分配问题：奖励往往延迟出现（如下棋仅在对局结束才有奖惩），智能体需要自行判断过往哪些动作带来了最终奖励或惩罚；
      • 存在部分可观测性：当下的观察可能无法体现环境完整状态，智能体需要结合历史信息判断处境；
      • 需权衡探索与利用：选择沿用当前最优策略获取稳定回报，还是尝试新动作、探索潜在更优策略。

    4. 典型问题分类（按环境特征划分）

      • 马尔可夫决策过程：环境状态可被智能体完整观测；
      • 上下文赌博机：环境状态不依赖历史动作；
      • 多臂赌博机：不存在环境状态，仅有一组收益未知的可选动作。

    5. 主要应用场景 广泛用于机器人、对话系统、游戏AI、物理仿真模拟等需要智能体自主决策、持续和环境互动的场景。

    1. 知识点梳理

    1. 数理统计：领域最早的理论根基 人类很早就运用统计思维做估算，中世纪已出现取均值、修剪均值的实践；后续诞生伯努利分布、高斯分布等经典概率理论，高斯提出的最小均方算法沿用至今。近代罗纳德·费舍尔完善统计理论，提出线性判别分析，发布的鸢尾花卉数据集成为经典测试数据，同时历史案例也警示数据科学需重视伦理问题。
    2. 信息论与计算理论：人工智能的理论支撑 香农创立的信息论为机器学习提供底层理论；图灵提出图灵测试，首次界定机器智能的判断标准，奠定人工智能计算理论基础。
    3. 神经科学与心理学：神经网络的设计灵感 赫布提出神经元强化学习的赫布学习理论，是早期神经网络算法的源头；现代神经网络借鉴生物神经元结构，形成两大核心机制：线性与非线性单元逐层堆叠、依靠反向传播统一调整网络参数。
    4. 神经网络发展低谷 1995—2005年神经网络发展受阻，受算力、数据两大条件限制；同期核方法、决策树、图模型等传统统计方法因训练高效、理论成熟，成为行业主流。

      1. 算力不足：当时硬件计算能力薄弱，训练神经网络耗时极长、成本高昂；
      2. 数据稀缺：大规模数据集尚未普及，难以发挥神经网络的优势。

    5. 应用阶段：传统机器学习在数据、算力匮乏的阶段成为主流；深度学习在大数据、高算力时代实现爆发式发展。

    总结

    2010年前后，沉寂多年的深度学习迎来快速复兴，其崛起并非全新理论的突然诞生，而是早期神经网络算法在数据、硬件、技术与工具多重条件成熟后的全面落地，整体发展脉络可分为五大核心部分：

    1. 复兴的两大核心动因 早期神经网络受限于数据稀缺与算力不足，发展一度停滞。随着互联网、智能传感器普及，大规模数据集得以积累；同时存储硬件迭代、GPU大规模商用，高性能算力成本大幅降低，两大瓶颈被彻底打破，为深度学习发展奠定了底层基础。

    2. 软硬件的演进规律 从1970年至2020年，数据集规模、设备浮点运算能力呈指数级增长，但内存容量的提升速度远跟不上数据与算力的发展。这一现状倒逼模型优化内存使用效率，也推动行业研究从传统线性模型、核方法，转向具备非线性表达能力的深度神经网络。

    3. 经典算法重新应用 多层感知机、卷积神经网络、长短期记忆网络、Q学习等数十年前提出的经典算法，依托海量数据与强大算力重新发挥价值，成为现代深度学习与深度强化学习的基础骨架。

    4. 近十年关键技术创新 这一阶段涌现出多项标志性技术，全方位补齐模型能力与训练效率：通过Dropout注入噪声缓解过拟合问题；注意力机制在不增加参数量的前提下优化长序列任务效果；多阶段设计赋予模型迭代推理能力；生成对抗网络实现图像、音频等复杂非结构化数据的自主生成；并行与分布式训练突破单卡算力限制，扩大训练批次、将大型模型训练时长从数天压缩至数分钟；并行计算也同步赋能强化学习，使游戏AI、仿真模拟等领域实现超越人类的表现。

    5. 深度学习框架迭代普及 深度学习工具历经三代演进：第一代以Caffe、Torch为代表，第二代以TensorFlow、MXNet为主流，第三代PyTorch等命令式框架语法简洁、易用性大幅提升。框架的不断简化极大降低了开发门槛，让深度学习从实验室技术转变为各行各业可便捷使用的通用工具，进一步推动了技术的传播与落地。

    总体而言，深度学习的发展是理论积淀、数据红利、硬件升级、技术创新、开源生态共同作用的结果，也让机器学习的应用边界与能力实现了跨越式拓展。

    6. 核心内容解析

    1. 知识点梳理

    1. 多级表示学习 深度学习的核心本质是多级表示学习。模型通过多层网络对数据进行逐层转换，靠近输入的网络层学习图像像素、语音波形等低级细节特征，靠近输出的网络层提炼高级抽象概念，依靠多层级特征表达完成任务，这也是“深度”的由来。

    2. 端到端训练 传统机器学习需要将流程拆分，先由人工完成特征工程，再将特征输入浅层模型训练。深度学习采用端到端训练，直接对整套系统进行联合调优，用网络层自动替代人工特征提取环节，不仅减少人工成本，还打破了计算机视觉、语音、自然语言处理等领域的技术壁垒。

    3. 模型形态转变 数据稀缺的阶段，主流使用参数模型，依靠人为简化假设来构建模型；进入大数据时代后，模型逐步转向非参数模型，不再依赖强先验假设，能够更精准地拟合真实数据分布，代价是模型可解释性有所下降。

    4. 优化与研究风格 深度学习普遍面对非凸、非线性的复杂优化问题，行业不再执着于寻找理论最优解，而是接纳次优解；研究风格偏向经验主义，优先通过实践验证算法效果，再补充理论解释，推动算法快速迭代更新。

    5. 开源生态建设 深度学习学术界与产业界高度联动，大量优秀算法、网络模型、训练框架对外开源共享。统一的工具链让复杂模型的搭建、训练变得简单，极大降低了入门门槛，推动技术大范围普及。

    受数据规模影响，主流模型形态发生明显转变：数据稀缺时，多使用依赖人为先验假设的参数模型；进入大数据时代后，非参数模型成为主流，这类模型弱化人工假设、更贴合真实数据分布，缺点是可解释性有所降低。

    在优化与研究风格方面，深度学习常面对非凸、非线性的复杂问题，行业不再一味追求理论最优解，而是接纳次优解；整体研究偏向经验主义，优先通过实践验证算法效果，再补充理论分析，有效加快了算法迭代速度。

    此外，深度学习领域具备浓厚的开源共享氛围，学术界与产业界开放各类算法、预训练模型和开发框架，极大降低了技术学习与落地的门槛，推动技术快速普及。

    整体而言，深度学习在学习逻辑、训练流程、模型选择、研究思路上完成了范式升级，叠加大数据与开源生态的助力，形成了独有的技术体系与发展优势。
